# PRD: VideoCraft Agent AI 营销短视频生成助手

## 1. 背景

目标岗位强调 AI 视频方向、文本到视频自动化、素材库对接、FFmpeg、Agent 工作流和业务可用性。直接训练或深度调优文生视频模型成本高，不适合作为短期求职项目。

VideoCraft Agent 选择更可落地的路线：用 Dify Workflow 承担 AI 脚本和分镜生成，用 FastAPI 承担上传、任务管理和服务编排，用本地素材库模拟企业素材库，用 FFmpeg 完成实际视频合成。

## 2. 产品目标

构建一个本地可运行的 AI 营销短视频生成工具。用户输入产品信息并上传素材后，系统自动生成一条 15 秒左右的竖屏营销视频。

核心目标：

- 跑通完整链路：文本需求 / 上传素材 -> Dify 脚本 -> FastAPI 编排 -> FFmpeg 合成 -> 预览和下载。
- 体现 Agent 思路：把 AI 生成、素材匹配、视频合成、格式转换拆成清晰节点。
- 体现业务理解：围绕营销短视频、产品介绍视频做设计。
- 体现工程能力：有前端、有后端、有状态、有错误处理、有文件输出。
- 适合学习和面试：每一层职责都能单独解释和替换。

## 3. 目标用户

主要用户：

- 运营人员：需要批量制作产品宣传视频。
- 小团队老板：没有专职剪辑，但需要快速做短视频。
- 产品或市场同事：需要把产品卖点变成短视频脚本和成片。

学习项目中的实际用户：

- 项目作者本人，用它学习 Dify、FastAPI、Python、FFmpeg 和业务型 AI 应用。
- 面试官，用它判断候选人是否能把 AI 能力落到业务工具里。

## 4. 用户问题

当前人工制作营销短视频通常需要：

- 手写脚本。
- 找素材。
- 剪辑。
- 加字幕。
- 转格式。
- 反复调整比例和时长。

这些步骤对非专业剪辑人员门槛高，对批量生产场景效率低。VideoCraft Agent 要解决的是“先产出一版能用的视频草稿”，让人只需要做少量检查和修改。

## 5. MVP 范围

MVP 只做最小闭环：

1. 用户在 Python 前端填写产品名称、目标人群、核心卖点、视频风格、目标平台、视频时长。
2. 用户可上传视频素材或产品图片。
3. FastAPI 保存上传文件并创建任务。
4. FastAPI 调用 Dify Workflow 生成 3 段式短视频脚本 JSON。
5. FastAPI 用 Pydantic 校验 Dify 返回结果。
6. 系统根据脚本关键词从上传素材或本地素材库选择视频片段。
7. FFmpeg 将片段处理成统一 9:16 竖屏比例。
8. FFmpeg 拼接视频并添加字幕。
9. 可选添加背景音乐。
10. 输出 mp4 文件，前端提供预览和下载。
11. 提供一个单独的视频格式转换功能。

## 6. 非目标

MVP 阶段不做：

- 不训练文生视频模型。
- 不接真实商业素材库 API。
- 不做用户登录。
- 不做复杂剪辑时间线编辑器。
- 不做自动配音。
- 不做视频内容审核。
- 不保证商用版权合规。

这些可以作为后续版本规划。

## 7. 核心流程

用户流程：

```text
打开页面
  -> 填写产品信息
  -> 上传素材
  -> 点击生成视频
  -> 查看生成状态
  -> 预览 Dify 脚本和成片
  -> 下载 mp4
  -> 可选转换格式
```

系统流程：

```text
FastAPI 接收输入和文件
  -> 保存任务和上传素材
  -> 调用 Dify Workflow
  -> 校验脚本 JSON
  -> 匹配上传素材或本地素材
  -> FFmpeg 标准化素材尺寸
  -> FFmpeg 拼接视频
  -> 添加字幕和音乐
  -> 输出文件
```

## 8. 功能需求

### 8.1 需求输入

用户可以填写：

- 产品名称。
- 目标用户。
- 核心卖点。
- 视频风格。
- 目标平台。
- 视频时长。
- 输出格式。
- 上传素材文件。

默认值：

- 视频时长：15 秒。
- 分镜数量：3 段。
- 平台：抖音。
- 画面比例：9:16。
- 输出格式：mp4。

### 8.2 Dify 脚本生成

Dify Workflow 应生成结构化脚本：

- index: 分镜序号。
- duration_seconds: 每段时长。
- subtitle: 字幕。
- voiceover: 旁白。
- visual_keywords: 画面关键词。
- source_hint: 素材来源建议。

脚本要求：

- 只输出 JSON 或由 End 节点输出 JSON 字段。
- 每段字幕短，适合短视频。
- 第一段负责吸引注意。
- 第二段负责放大痛点和卖点。
- 第三段负责行动号召。

### 8.3 素材匹配

系统根据 `visual_keywords` 匹配素材：

- 优先使用用户上传素材。
- 上传素材不足时使用本地素材库。
- 无匹配时使用 default.mp4。

MVP 采用关键词映射，不做向量检索。

### 8.4 视频合成

系统调用 FFmpeg 完成：

- 裁剪每段素材到指定时长。
- 转为 1080x1920 竖屏。
- 拼接多个片段。
- 添加字幕。
- 添加背景音乐。
- 输出 mp4。

### 8.5 格式转换

用户可以上传或选择已有视频，将其转换为：

- mp4。
- mov。
- webm。
- gif。

### 8.6 任务状态

页面展示状态：

- 等待输入。
- 已保存上传素材。
- 正在调用 Dify。
- 脚本已生成。
- 正在匹配素材。
- 正在合成视频。
- 正在导出文件。
- 完成。
- 失败。

### 8.7 错误处理

至少处理：

- Dify 调用失败。
- Dify 返回 JSON 解析失败。
- 上传文件格式不支持。
- 素材文件不存在。
- FFmpeg 未安装。
- FFmpeg 合成失败。
- 输出目录不存在。

错误信息要给用户可理解的提示，同时在日志中保留技术细节。

## 9. 成功指标

MVP 成功标准：

- 能通过一个页面生成完整 15 秒左右的视频。
- 能调用 Dify 生成结构化脚本。
- 能展示生成脚本。
- 能完成素材匹配。
- 能生成带字幕的竖屏 mp4。
- 能完成至少 1 种格式转换。
- 出错时不是直接崩溃，而是给出明确原因。

学习成功标准：

- 能解释为什么让 Dify 做 AI 工作流。
- 能解释为什么让 FastAPI 做主控服务。
- 能解释 FFmpeg 在项目中的作用。
- 能说清楚如何从业务需求变成技术模块。
- 能说出后续如何接入开源文生视频模型。

## 10. 版本规划

V0.1 AI 脚本 + FFmpeg 成片闭环：

- 在 Dify 创建 Workflow 应用。
- Dify 输出固定 JSON。
- FastAPI 调用 Dify 并校验脚本。
- 用上传素材或默认素材生成 9:16 mp4。

V0.2 前端上传页面：

- Python 前端输入产品信息。
- 支持上传素材。
- 状态展示。
- 视频预览。
- 下载按钮。

V0.3 稳定性增强：

- Dify 失败兜底。
- JSON 解析失败处理。
- FFmpeg 错误日志。
- 上传文件校验。

V0.4 视频工具增强：

- 增加格式转换。
- 增加背景音乐选择。
- 增加平台模板。

V1.0 面试版：

- 完成录屏或截图。
- 完成项目复盘。
- 完成简历描述。
